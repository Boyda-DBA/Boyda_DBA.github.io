U.S Consumer Finance Complaints Analysis

An ETL Project using the U.S Consumer Finance Complaints dataset from Kaggle.

The Process

Begining with the raw data in CSV format looked through the columns and data types to build an ERD for the structure of the database.

ERD


This structure identified the main entities at play in the dataset. The structure of the product and issues tables are self refferencial using a parent ID to efficiently represent the heirarchical data stored in the sub_product and sub_issue columns.

Data Dictionary

Product Table

Field Name Data Type Description Required
product_id int4 The primary key of the product table Yes
product text The product description Yes
parent_product_id int4 The foreign key to the parent product for sub_products No

Issue Table

Field Name Data Type Description Required
issue_id int4 The primary key of the issue table Yes
issue text The issue description Yes
parent_issue_id int4 The foreign key to the parent issue for sub_issues No

Company Table

Field Name Data Type Description Required
company_id int4 The primary key of the company table Yes
company text The name of the company Yes

Complaint Table

Field Name Data Type Description Required
complaint_id int4 The primary key of the complaint table Yes
consumer_complaint_narrative text The detailed complaint filed by the consumer No
company_public_response text The response from the company, for public view No
submitted_via text The medium through which the complaint was submitted Yes
date_received date The date and time the complaint was received Yes
date_sent_to_company date The date and time the complaint was sent to the company Yes
company_response_to_consumer text The response from the company to the consumer Yes
timely_response bool Whether the company responded in a timely manner Yes
consumer_disputed bool Whether the consumer disputed the response or not Yes
complaint_state text The state from which the complaint was filed No
complaint_zip int4 The ZIP code from which the complaint was filed No
complaint_tags text Tags related to the complaint No
consumer_consent_provided text If the consumer provided consent for complaint share No
issue_id int4 The foreign key to the issue related to the complaint Yes
product_id int4 The foreign key to the product related to the complaint Yes
company_id int4 The foreign key to the company the complaint is against Yes
